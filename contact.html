Best reasons to reach out

  • A calculation or workflow looks wrong.
  • You found a bug on desktop or mobile.
  • You want support for a box style or feature that is not there yet.
  • You have a real-world build result that would help improve the calculator.

Contact method

For now, the simplest contact path is through the project repository or the site owner’s public channels tied to this project.

If you later want a cleaner public support flow, the next step would be adding a dedicated email address or a lightweight contact form hosted off GitHub Pages.

What helps in a bug report

  • Your enclosure type and cabinet style.
  • The dimensions or settings you entered.
  • What you expected to happen.
  • What actually happened.
  • Your device and browser if the issue looks layout-related.
